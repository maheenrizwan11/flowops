WORKFLOW 2: Review and Decision Process
1. Reviewer logs in. 
2. System verifies Reviewer role. 
3. Reviewer views Submitted requests. 
4. Reviewer opens a request. 
5. Reviewer changes status to Under Review. 
6. System logs action. 
7. Reviewer approves or rejects request. 
8. System updates status to Approved or Rejected. 
9. System records decision in audit logs. 
10. Request is marked Closed.

WORKFLOW 3: Request Reassignment and Escalation
1. Reviewer logs in to FlowOps.
2. System verifies Reviewer role.
3. Reviewer views the list of Submitted or Under Review requests.
4. Reviewer identifies a request that requires escalation or reassignment (e.g., missing info, high priority, or complex dispute).
5. Reviewer selects a new assignee (another Reviewer or Admin) and reassigns the request.
6. System updates the request status to Escalated and records the reassignment action in audit logs.
7. The new assignee receives a notification about the reassigned request.
8. New assignee reviews the request and continues processing (approval or rejection).
9. System logs all reassignment and escalation actions with user, role, timestamp, and request ID.
